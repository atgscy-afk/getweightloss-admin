# Chats

## Overview

**Chats** shows all conversations between members and support staff. It's a record of what's been discussed and helps you avoid repeating questions.

## Viewing Chat Threads

1. Go to **Chats** → **Chat List**
2. See all chat threads, newest first
3. Filter by:
   - Member name
   - Date
   - Status (open, resolved, waiting for member)

## Reading a Chat

Click a chat thread to see the full conversation:

- **Participants** — who's been chatting
- **Messages** — the full conversation history
- **Timestamps** — when each message was sent
- **Attachments** — files member or staff shared

## Responding in a Chat

1. Scroll to the bottom
2. Type your response
3. Click **Send**

The member gets a notification. This conversation is automatically logged on their member profile.

## Chat Status

| Status | Means |
|--------|-------|
| **Open** | Actively being discussed |
| **Waiting on Member** | You've replied; waiting for them to respond |
| **Resolved** | Issue is sorted; chat can be closed |
| **Closed** | No further action needed |

After you respond, mark status as **Waiting on Member** so everyone knows it's the customer's turn.

When the issue is resolved:
1. Click **Mark as Resolved**
2. Add a summary (optional)
3. Chat is archived

## Common Chat Issues

### "Member asked a question I don't know how to answer"

1. Post a helpful response (even if partial)
2. Mark status as **Waiting on Member** or **Open**
3. Tag another team member (`@manager-name`) to review
4. They'll see it and reply

### "Member hasn't responded in a week"

1. Check the **date of last message**
2. Send a follow-up: "Just checking in — let us know if you need anything else!"
3. Mark as **Waiting on Member**

### "Chat is resolved but they message again later"

1. Chat auto-reopens when a new message arrives
2. Respond as normal
3. Mark resolved again when done
